Ordering¶

Generally, API results are orderable by the local attributes defined in the REST data model.

Intro¶

For example, given the data model:

User Datatype
id id
created datetime
name string
account_type ref

Order on name in ascending order:

GET /users/?order_by=name¶

Order on name in descending order:

GET /users/?order_by=-name¶

Order on multiple fields:

GET /users/?order_by=created,name¶

Ordering Objects by Meta values¶

Many objects support a Meta interface for generic key/value like storage. You may use a value that exists on Meta with the order_by URL parameter as well.

Warning

Currently, only one meta__ field can be supplied in the order_by.

For example:

GET /devices/?order_by=-meta__dim¶
