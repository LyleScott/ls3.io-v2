Description¶

Gateway Lite is a special type of Gateway aimed at residential and small business Users. Gateway Lite should have a custom field named installation_code to allow the Cloud API to talk directly to it. This installation code is usually printed on the back of the Gateway.

User Registration¶

Obtaining an Integration Key¶

Integration Key is a token associated with each Gateway Lite Provider (Partners with is_gateway_lite_provider=True). They can be used to allow Users to self register and manage Gateway Lite Devices.

GET /partners/(id)/integration_key/¶
Status Codes:
  • 200 OK – Response served successfully.
  • 404 Not Found – The Partner is not a Gateway Lite provider (is_gateway_lite_provider=True).

Example request:

GET /partners/(id)/integration_key/ HTTP/1.1
Host: api.gooee.io
Accept: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "integration_key": "3878732iydjdahdajhd",
}

Registering as a Gateway Lite User¶

POST /integrations/gateway_lite/users/register/¶
JSON Parameters:
 
  • integration_key (string) – An integration key provided by the Partner to allow User’s registration.
  • first_name (string) – User’s first name (required)
  • last_name (string) – User’s last name (required)
  • email (string) – User’s email (required)
  • password (string) – Set a password for the User
  • password_confirm (string) – Confirm User’s password
Status Codes:
  • 201 Created – user was created successfully
  • 400 Bad Request – parameters are missing or incorrect

Example request:

POST /integrations/gateway_lite/users/register/ HTTP/1.1
Host: api.gooee.io
Accept: application/json

{
  "integration_key": "3878732iydjdahdajhd",
  "first_name": "Bob",
  "last_name": "Bobsen",
  "email": "bob.bobsen@email.com",
  "password": "somepassword",
  "password_confirm": "somepassword"
}

Example response:

HTTP/1.1 201 CREATED
Content-Type: application/json

{
    "detail": "User account registered successfully. Please check your email to activate it."
}

Activating a Gateway Lite User¶

Once a User has registered, she/he will receive an email with a 6 digit token and must use this endpoint to activate the account and login.

POST /integrations/gateway_lite/users/activate/¶
JSON Parameters:
 
  • activation_token (string) – the activation token sent to the user (required)
Status Codes:
  • 200 OK – user was activated successfully
  • 400 Bad Request – parameters are missing or incorrect
  • 404 Not Found – activation_token not found

Example request:

POST /integrations/gateway_lite/users/activate/ HTTP/1.1
Host: api.gooee.io
Accept: application/json

{
  "activation_token": "387873"
}

Example response:

HTTP/1.1 200 OK

Acquiring a Gateway Lite¶

Acquiring a Gateway Lite is the process of getting it from inactive commission state to customer_provisioned, making the authenticated User the owner of the Gateway. This process is done by providing the euid located on the back of the Gateway as raw data or scanning the QR code on it.

POST /integrations/gateway_lite/acquire¶
Status Codes:
  • 200 OK – The device was acquired correctly.
  • 400 Bad Request – Invalid request data.
  • 400 Bad Request – Device already acquired.
  • 404 Not Found – Device not found.

Request Model

Key Datatype Notes
euid string required

Example request:

POST /integrations/gateway_lite/acquire HTTP/1.1
Host: api.gooee.io
Content-Type: application/json

{
      "euid": "0200000100001274"
}

Example responses:

HTTP/1.1 200 OK
Content-Type: application/json

{
      "uuid": "120e54bb-0fdc-4929-a7f4-7fab08cb0469",
      "euid": "0200000100001274",
      "access_url": "0200000100001274-88d198253b85f1a35282.p2p.gooee.com"
}

Releasing a Gateway Lite¶

Releasing is the process of un-scoping (removing ownership) of a Gateway Lite from the Customer which brings it back to the inactive state.

PUT /devices/(ID)/release/¶
PATCH /devices/(ID)/release/¶
Status Codes:
  • 200 OK – The Gateway was released successfully.
  • 404 Not Found – The Gateway cannot be found.

Example request:

PUT /devices/867f431a-601a-480f-a590-570ebc1c3fdb/release/ HTTP/1.1
Host: api.gooee.io
Content-Type: application/json

Example response:

HTTP/1.1 200 OK
Content-Type: application/json

{
  "detail": "The Gateway was released successfully."
}

Fingerprint Management¶

Fingerprints are used by Gateway Lite to allow User’s mobile devices to authenticate on the Gateway Lite to control itself and it’s associated Devices. They are represented by a 32 characters hexadecimal string.

Data Model¶

Key Datatype Notes
id ref read-only
fingerprint string required, max_length=32, unique, base16 value
user ref write-only, defaults to the logged User
description string  

Create a Fingerprint¶

POST /integrations/gateway_lite/fingerprints¶
Status Codes:
  • 201 Created – Fingerprint was created correctly.

Example request:

POST /integrations/gateway_lite/fingerprints HTTP/1.1
Host: api.gooee.io
Content-Type: application/json

{
  "fingerprint": "e66db5dc25c4423da290d42a5b57af21",
  "description": "Iphone 7s Plus",
}

Example response:

HTTP/1.1 201 CREATED
Content-Type: application/json

{
  "id": "867f431a-601a-480f-a590-570ebc1c3fdb",
  "fingerprint": "e66db5dc25c4423da290d42a5b57af21",
  "description": "Iphone 7s Plus"
}

Resyncing Fingerprints¶

Resync all Fingerprints scoped to your User.

POST /integrations/gateway_lite/fingerprints/resync¶
Status Codes:
  • 200 OK – Synchronizing 3 fingerprint(s).

Example request:

POST /integrations/gateway_lite/fingerprints/resync HTTP/1.1
Host: api.gooee.io
Content-Type: application/json

Resync specific Fingerprints scoped to your User by specifying an array of Fingerprints in the request payload. Alternatively, you can use their IDs if hyphens are omitted.

POST /integrations/gateway_lite/fingerprints/resync¶
Status Codes:
  • 200 OK – Synchronizing 2 fingerprint(s).

Example request:

POST /integrations/gateway_lite/fingerprints/resync HTTP/1.1
Host: api.gooee.io
Content-Type: application/json

{
  "fingerprints": ["f641649b1a2a4f3caee3c40c16b9806b", "ebd70e23ebd1439fb325bda663c52266"]
}

Importing Gateway Lite Devices¶

An endpoint for uploading a CSV file containing device information that will be processed into individual Gateway Lite devices that can be acquired by consumers.

Data Model¶

Key Datatype Notes
id ref read-only
created datetime read-only
modified datetime read-only
csv_file file required
product_id UUID required
status enum read-only
customer_scopes array(ref) see Scope
partner_scopes array(ref) see Scope
error_message string read-only
partner_id UUID read-only
tags array(str) max_length=50 per tag

Complex Constraints¶

You must upload a CSV file with ; delimiters and containing a header of: "Product Serial Number";"FW Code";"Nabto DeviceId";"InstallCode"

POST /integrations/gateway_lite/import¶
Status Codes:
  • 201 Created –

Example request:

Host: api.gooee.io
Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW

Content-Disposition: form-data; name="csv_file"; filename="gateway_lite_devices.csv"
Content-Type: text/csv
