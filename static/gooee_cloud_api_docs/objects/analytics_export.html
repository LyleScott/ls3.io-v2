DescriptionÂ¶

An Analytics Export allows you to export all of the results of an Analytics Query to a CSV file that is available to download. This makes it easier to consume large result sets and the export file can be referenced for a period of time.

Note

You could accomplish the same with manual Analytics History API requests by paginating through all results and converting column-major-order JSON to a CSV file.

Status WorkflowÂ¶

When you create an export, you are actually scheduling an export to execute as soon as possible. This allows us to handle large data exports and allows the user to consume all results of a query with the least amount of complexity.

As the export starts and eventually completes or fails, the status field is updated accordingly. This means that API consumers must poll for the desired status to occur.

When the status becomes complete, a url is then populated on the resource. This URL is a direct link to a downloadable CSV file that will expire in 1 hour from the time it was generated.

Export File StructureÂ¶

The downloaded file will be in CSV text (text/csv) format.
