Description¶

Buildings are composed of a hierarchy of Spaces. They belong to a Customer and have a physical Location.

Data Model¶

Key Datatype Notes
id ref read-only
created datetime read-only
modified datetime read-only
name string required, max_length=100
description string  
customer ref required, write-once
spaces array(ref)  
physical_spaces array(ref) read-only
virtual_spaces array(ref) read-only
location location  
timezone enum default=UTC, see Timezones
currency enum default=USD, see Currency
auto_dst boolean default=true, Automatic Daylight Savings
last_activity datetime read-only
meta array(meta)  
data_service_id string read-only
anx_dimensions array(str) see Dimensions
customer_scopes array(ref) see Scope
partner_scopes array(ref) see Scope
tags array(str) max_length=50 per tag
reactor_version string max_length=64, s3-generated version e.g. ‘unknown’, write-only
rule_version string read-only, options=[v1, v2], default=v2

Device Counts¶

Device counts (online/total) are supported on an opt-in basis via the “_annotate” filter. See: Device Counts for reference.

Operating Hours¶

Every Building may have operating hours to define the times that it will be open. By default Buildings will have empty operating hours [].

Data Model¶

Key Datatype Notes
name string required, unique, max_length=100
effective_date date required, unique, format=`YYYY-mm-dd`
time_schedule object see Time Schedule

Time Schedule¶

Data Model¶

Key Datatype Notes
day_of_week string required, choices=[mon,tue,wed,thu,fri,sat,sun]
start time required, format=`HH:MM`
end time required, format=`HH:MM`

Complex Constraints¶

start and end for a single day must not match or overlap.

Example¶

"operating_hours" : [
  {
    "effective_date": "2017-07-01",
    "name": "Summer Working Hours",
    "time_schedule": [
      {
        "day_of_week": "mon",
        "start": "09:00",
        "end": "17:30"
      },
      {
        "day_of_week": "tue",
        "start": "09:00",
        "end": "17:30"
      },
      {
        "day_of_week": "wed",
        "start": "09:00",
        "end": "17:30"
      },
      {
        "day_of_week": "thu",
        "start": "09:00",
        "end": "17:30"
      },
      {
        "day_of_week": "fri",
        "start": "09:00",
        "end": "17:30"
      },
      {
        "day_of_week": "sat",
        "start": "08:00",
        "end": "12:00"
      }
    ]
  }
]

Electrical Fee Schedule¶

An electrical fee schedule may be added to a building. The “name”, “effective_date” and “base_rate” fields are required and establish a default rate for the building. Exceptions to the default schedule may be added in the “time_dependent_rates” field.

By default, every Building has an empty electrical fee schedule: [].

Data Model¶

Key Datatype Notes
name string required, unique, max_length=100
effective_date date required, unique, format=`YYYY-mm-dd`
base_rate number required, > 0
time_dependent_rates object see Time Dependent Rates

Complex Constraints¶

effective_date must be unique for the Building.

name must be unique for the Building.

Example¶

"electrical_fee_schedule" : [
  {
    "effective_date": "2016-01-01",     // YYYY-mm-dd
    "base_rate": 150,                   // Positive number
    "name": "Winter 2016",
    "time_dependent_rates": [
      {
        "day_of_week": "mon",           // (mon,tue,wed,thu,fri,sat,sun)
        "start": "07:00",               // 24 hour time (HH:MM)
        "end": "18:30",
        "rate": 200
      },
      {
        "day_of_week": "tue",
        "start": "07:00",
        "end": "18:30",
        "rate": 200
      },
      {
        "day_of_week": "wed",
        "start": "07:00",
        "end": "18:30",
        "rate": 200
      },
      {
        "day_of_week": "thu",
        "start": "07:00",
        "end": "18:30",
        "rate": 200
      },
      {
        "day_of_week": "fri",
        "start": "07:00",
        "end": "18:30",
        "rate": 200
      }
    ]
  }
]
