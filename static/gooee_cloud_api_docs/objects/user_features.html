User Features¶

Description¶

Features are used to describe functionality which is enabled/disabled for a particular User. We return the features in the API’s Auth Response. The following would be the relevant part of the API Auth Response which lists the status of two sample features named analytics and history:

{
    "features": {
        "analytics": true,
        "history": false
    }
}

Data Model¶

Key Datatype Notes
features array(obj) JSON Field

Complex Constraints¶

Only Gooee Users are allowed to set feature flags.

API Endpoints¶

Update features¶

PUT /users/(id)/features¶
Status Codes:
  • 200 OK –

Partially Update features¶

PATCH /users/(id)/features¶
Status Codes:
  • 200 OK –
