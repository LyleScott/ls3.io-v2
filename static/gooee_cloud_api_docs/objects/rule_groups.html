DescriptionÂ¶

Rule Groups are provided as a mechanism for linking related Property-based Rules and enforcing consistency between linked Rules (see Complex Constraints and Property-based Rules docs). This feature exists purely to satisfy frontend apps that have a need for such constraints.

id ref read-only

name string required, max_length=100
description string  
tags array(str) max_length=50 per tag
property_rules array(ref) required, create-only, min_length=2, Complex Constraints
space ref required, Complex Constraints
enabled boolean required, Complex Constraints
execution_origin enum required, options=["cloud", "gateway"], Complex Constraints

Complex ConstraintsÂ¶

  • name and space must form a unique pair.
  • All property_rules must have the same values as the Rule Group for space, enabled, and execution_origin.
  • Changes to enabled or execution_origin are propagated to all property_rules.
