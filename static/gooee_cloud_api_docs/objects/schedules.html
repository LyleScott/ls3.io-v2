Description¶

Schedules are a group of Operations that happen at a specific time.

Data Model¶

Key Datatype Notes
id ref read-only
created datetime read-only
modified datetime read-only
name string required
description string  
time_of_day string HH:MM:SS in 24 hour format
day_of_week array(str) Ex: [“mon”,”tue”,”wed”,”thu”,”fri”,”sat”,”sun”]
building ref Complex Constraints
spaces array(ref) read-only
devices array(ref) read-only
operations array(obj) ref required, Complex Constraints
customer_scopes array(ref) see Scope
partner_scopes array(ref) see Scope
last_execution datetime read-only
execution_count number read-only

Complex Constraints¶

A Schedule must have a Building or a set of Operations containing one or more Spaces or Devices.

A Schedule must have at least one Operation.

All Operation Spaces and Devices must be within the same Building.

All Operation Device Actions must be supported by the target Devices.

All Operation Devices must have a Building.

An individual Operation can only be used against Spaces or Devices.

Operations¶

When the conditions are met, a Schedule executes its Operations. Operations are a set of Actions applied over a set of Spaces or Devices.

   "building": "{{BUILDING_ID}}"
