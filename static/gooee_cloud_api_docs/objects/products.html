Description¶

Products provide the fundamental information used for interacting with Devices. They provide information regarding capabilities and technical data available. They can have a product_owner which is the Partner who creates the product and a manufacturer being the Partner with is_manufacturer=True representing who made the product, so that if the manufacturer has an account in our system they can claim ownership of the product.

Note

There are Product Specs that, when set to true, allow for custom constraints on the Product. This puts a limit on the Action targeting a Device, so that its value can only fall within the given constraints.”

Complex Constraints¶

SKUs must be unique per Manufacturer.

Only Partner with is_manufacturer=True are valid selections for a Product’s manufacturer.

Only Partner users or Gooee users can create Products.

Only Gooee users can change the owner of a Product.

Only Gooee users can edit specs of Product whose state is ‘active’.

Product Specs¶

The Product specs are populated with default values from the Product type when the Product is created and they can only be edited while the Product is in a draft state. Product specs cannot be partially updated — the whole specs object gets replaced with new values on a PUT or PATCH. If Product has been activated with incorrect specs contact cloud-backend@gooee.com .

Key Datatype Notes
name string required, max_length=100, Complex Constraints.
display_name string max_length=100, Complex Constraints.
value number, string, array, boolean required, Complex Constraints.
modified datetime Complex Constraints.

Complex Constraints¶

  • name is limited to letters, numbers, and underscore (_). It must be a unique spec name from the Product type.
  • display_name is not required, if is not provided it will come from the Product type.
  • value is not required, if is not provided it will be set to a default one according to the datatype defined for the spec on the Product type. The value is validated against the definition on the Product type.
  • modified is read only, is set when the value changes from default one to a user provided value.

Example specs:

[
    {
      "name": "ctrl_dimmable",
      "display_name": "Dimmable Support",
      "modified": null,
      "value": true
    },
    {
      "name": "bcn_ibeacon",
      "display_name": "iBeacon Support",
      "modified": null,
      "value": true
    },
]

Status Codes:
  • 200 OK –

Delete a Product¶

Only owner of the Product may delete its products.
DELETE /products/(id)/¶
Status Codes:
  • 204 No Content –

Activate a Product¶

Only owner of the Product may activate its products.
PUT /products/(id)/activate¶
Status Codes:
  • 204 No Content – Product was activated successfully.
  • 400 Bad Request – Product is already active.
  • 412 Precondition Failed – All the specs need to be filled in order to activate this Product.
