  • Refactored the anx_dimensions field that is on some entities.

  • Analytics validation has changed to make it more explicit. - granularity=auto is no longer supported - start is now required - restrictions on too-fine or too-course granularity parameters has been lifted

6/11/2019

  • Modified Actions to be generic.
    • Instead of requiring a device/space, an Action needs a target_type and target_id.
    • Users may still use the old Data Model for the next 6 months, afterwards the fields will be deprecated.
  • Renamed available_on_rules field on Action Types to be rules_support
  • Added available_on field on Action Types to contain a list of target_type-s supported by the ActionType.
  • Removed available_on_devices and available_on_spaces fields on Action Types.
    • The data is now factored into the new available_on field, EX. "available_on": ["device", "space"]
  • Added the Blacklist and Wait Action Types.
  • Removed beacons_detected and beacons_exited Action Types
    • beacon_detected and beacon_exited are available for single Action requests.
  • Rules v2 operations field has been replaced by actions which is composed of an array of arrays.
    • The top level arrays are “Action groups” that contain a list of actions to be executed.
    • “Action groups” are executed sequentially but their containing actions are executed synchronously.
